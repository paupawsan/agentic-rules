# 🤔 批判的思考ルールプラグイン (日本語)

## 概要

**批判的思考ルール**プラグインは、AIエージェントに体系的な推論とエラー防止機能を付与します。このプラグインは、仮定の検証、証拠の評価、論理的一貫性の確保を実装し、より信頼性の高い意思決定を可能にします。

## 🎯 機能

批判的思考ルールは、エージェントが情報を批判的に評価し、論理的な決定を行うためのフレームワークを提供します：

- **仮定検証**: 自動的な仮定の挑戦と検証
- **証拠評価**: 提示された情報の強度と信頼性の評価
- **論理的整合性**: 矛盾の検出と解決
- **エラー防止**: 一般的な推論の落とし穴の回避

## ✨ 主な機能

### 批判的評価プロセス

#### 1. **仮定の特定と挑戦**
- 暗黙の仮定の認識
- 証拠による検証
- 代替説明の検討

#### 2. **証拠の評価**
- 情報の信頼性の評価
- バイアスの検出
- 証拠の強度と関連性の判断

#### 3. **論理的分析**
- 議論の構造分析
- 論理的誤りの特定
- 推論の妥当性検証

#### 4. **確率と不確実性の考慮**
- 確実性の適切な表現
- 限界と不確実性の認識
- 確率的推論の適用

### 品質保証メカニズム

#### 1. **一貫性チェック**
- 内部矛盾の検出
- 以前の声明との整合性検証
- 全体的な物語の一貫性

#### 2. **代替説明の検討**
- 複数の解釈の探求
- 最も単純な説明の優先
- 認知バイアスの回避

#### 3. **フィードバックループ**
- 修正からの学習
- 推論パターンの改善
- 継続的な精度向上

## 🚫 制限と考慮事項

### 認識的制約
- **計算複雑性**: 深い分析は計算コストが高い
- **コンテキスト依存**: ドメイン固有の知識が必要
- **曖昧性の処理**: 不確実性の適切な表現
- **文化的バイアス**: トレーニングデータの影響

### 実用的考慮
- **処理時間のトレードオフ**: 品質 vs 速度
- **過度な懐疑**: 分析麻痺の回避
- **バランスの必要**: 創造性と批判性のバランス

## 📝 セットアップと使用法

### 基本セットアップ
```bash
# 批判的思考ルールを有効化
"Enable critical thinking rules to ensure rigorous analysis and prevent errors."

# 信頼性基準を設定
"Apply critical thinking standards requiring [confidence level] for recommendations."
```

### 高度な構成
```bash
# カスタム評価基準を設定
"Configure critical thinking with evidence_threshold=0.8, bias_detection=enabled, and alternative_hypotheses=3."
```

### クエリ強化
```bash
# 批判的思考を促すプロンプト
"Before providing any recommendation, critically evaluate the assumptions, evidence, and potential biases. Consider alternative explanations and express appropriate confidence levels."
```

## ⚙️ 設定オプション

### 評価基準
```json
{
  "critical_thinking_rules": {
    "enabled": true,
    "evidence_threshold": 0.8,
    "bias_detection": true,
    "alternative_hypotheses": 3,
    "confidence_reporting": true
  }
}
```

### ドメイン固有設定
```json
{
  "domains": {
    "technical": {
      "rigor_level": "high",
      "evidence_types": ["empirical", "logical", "authoritative"]
    },
    "business": {
      "rigor_level": "medium",
      "evidence_types": ["data", "expert_opinion", "precedent"]
    }
  }
}
```

## 📊 パフォーマンス指標

### 品質メトリクス
- **正確性**: 推奨事項の正確性の割合
- **完全性**: 考慮された代替案の数
- **一貫性**: 内部矛盾のない応答の割合
- **信頼性**: 表現された確信度の正確性

### 効率性メトリクス
- **分析時間**: 批判的評価に要する時間
- **決定品質**: 改善された意思決定の測定
- **エラー削減**: 回避された誤りの数
- **ユーザ満足度**: 分析の有用性評価

## 🔧 トラブルシューティング

### 分析が遅すぎる
```
問題: 批判的思考プロセスが応答を遅らせる
解決: 分析の深さを調整、または条件付き適用
```

### 過度に懐疑的
```
問題: エージェントが決定を下さない
解決: 確信度のしきい値を調整、またはバランスモードを使用
```

### バイアス検出が不正確
```
問題: 誤ったバイアス警告
解決: トレーニングデータを更新、または検出感度を調整
```
