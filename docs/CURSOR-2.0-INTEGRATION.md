# Cursor 2.0 Multi-Agent Integration Guide

## Overview

This guide explains how to use the Agentic Rules Framework with Cursor 2.0's multi-agent system, including orchestrator and sub-agent configurations.

## Prerequisites

### Cursor 2.0 Setup

1. **Enable Nightly Channel**:
   - Go to `Cursor Settings` > `Beta`
   - Set Update Channel to `Nightly`
   - Restart Cursor

2. **Enable Agent Mode**:
   - Open Composer (`Cmd + I` or `Ctrl + I`)
   - Toggle to **Agent** mode (not just "Chat")

3. **Framework Initialization**:
   - Ensure Agentic Rules Framework is initialized
   - Verify `.agentic_initialized` marker exists
   - Check `bootstrap.json` for framework configuration

## Architecture

### Multi-Agent Structure

```
Orchestrator Agent (Primary)
├── Memory Specialist Agent
├── RAG Specialist Agent
├── Critical Thinking Specialist Agent
├── Documentation Specialist Agent
└── Testing Specialist Agent
```

### Framework Integration

- **Orchestrator**: Coordinates sub-agents using framework rules
- **Sub-Agents**: Specialized agents using specific framework modules
- **Rule-Based Coordination**: Uses `bootstrap.json` for orchestration
- **Memory Sharing**: Sub-agents share context through Memory Rules
- **Quality Assurance**: Critical Thinking Rules validate all outputs

## Agent Configuration

### Auto-Generation via setup.html

**All agent configurations are auto-generated by `setup.html`**. Users do not need to manually create these files.

#### Setup Process

1. **Open setup.html**: Open `setup.html` in your browser
2. **Select Deployment Target**: Choose "Cursor 2.0 Multi-Agent"
3. **Choose Language**: Select agent language (English, Indonesian, or Japanese)
4. **Select Rules**: Choose which framework modules to enable
5. **Generate Files**: Click "Generate Configuration Files"
6. **Copy to Project**: Copy generated `.cursor/agents/*.md` files to your project's `.cursor/agents/` directory

#### Generated Files

After generation, you'll have:

```
.cursor/agents/
├── orchestrator.md              # Primary orchestrator
├── memory-agent.md              # Memory operations specialist
├── rag-agent.md                 # Information retrieval specialist
├── critical-thinking-agent.md   # Quality assurance specialist
├── docs-agent.md                # Documentation specialist
├── test-agent.md                # Testing specialist
└── README.md                    # Quick start guide
```

**Note**: These files are **not committed to git** (already in `.gitignore`). Each user generates them for their own project.

### Agent Roles

#### Orchestrator Agent
- **Purpose**: Task decomposition and sub-agent coordination
- **Framework**: Uses all framework modules for coordination
- **Delegation**: Assigns tasks to specialized sub-agents
- **Synthesis**: Combines sub-agent outputs

#### Memory Specialist Agent
- **Purpose**: Context retrieval and knowledge persistence
- **Framework Module**: Memory Rules
- **Capabilities**: Memory storage, retrieval, knowledge graph integration

#### RAG Specialist Agent
- **Purpose**: Information retrieval and context optimization
- **Framework Module**: RAG Rules
- **Capabilities**: Semantic search, context optimization, knowledge graph queries

#### Critical Thinking Specialist Agent
- **Purpose**: Quality assurance and validation
- **Framework Module**: Critical Thinking Rules
- **Capabilities**: Error detection, assumption challenges, verification

#### Documentation Specialist Agent
- **Purpose**: Documentation generation and maintenance
- **Framework Modules**: All rules (enhanced documentation)
- **Capabilities**: Code docs, architecture docs, user guides

#### Testing Specialist Agent
- **Purpose**: Test creation and quality verification
- **Framework Modules**: Critical Thinking Rules, Memory Rules
- **Capabilities**: Unit tests, integration tests, coverage analysis

## Usage Patterns

### Basic Orchestration

1. **Open Composer**: Press `Cmd + I` (Mac) or `Ctrl + I` (Windows/Linux)
2. **Enable Agent Mode**: Toggle to Agent mode
3. **Use Plan Mode**: Press `Shift + Tab` for technical planning
4. **Global Context**: Use `@Codebase` for full project understanding
5. **Delegate Tasks**: Orchestrator automatically delegates to sub-agents

### Parallel Execution

Cursor 2.0 uses Git Worktrees for parallel agent execution:

1. **Launch Parallel Tasks**: Orchestrator delegates independent tasks
2. **Worktree Creation**: Cursor automatically creates worktrees
3. **Monitor Progress**: Check worktrees with `git worktree list`
4. **Apply Changes**: Use "Apply" button in UI to merge results

### Best-of-N Strategy

For critical tasks, use multiple models:

1. **Select Multiple Models**: Choose 2-3 models in model selector
2. **Submit Prompt**: Single prompt launches parallel agents
3. **Compare Results**: Toggle between agent "tabs"
4. **Select Best**: Choose most elegant solution
5. **Apply**: Merge selected solution

## Framework Coordination

### Rule-Based Delegation

The orchestrator uses framework rules for delegation:

- **Memory Tasks**: Delegated to Memory Specialist when context retrieval needed
- **Information Tasks**: Delegated to RAG Specialist for complex searches
- **Quality Tasks**: Delegated to Critical Thinking Specialist for validation
- **Documentation Tasks**: Delegated to Documentation Specialist
- **Testing Tasks**: Delegated to Testing Specialist

### Rule Interconnections

Framework rule interconnections guide agent coordination:

- **Error → Memory**: Error corrections stored in memory
- **RAG → Memory**: Optimized context enhances memory
- **Memory → RAG**: Session context improves RAG queries
- **Critical Thinking → RAG**: Quality validation for retrieved information
- **Memory → Critical Thinking**: User patterns inform assumption challenges

### Priority-Based Execution

Framework priorities determine agent activation:

- **High Priority**: RAG Rules, Critical Thinking Rules (always active)
- **Medium Priority**: Memory Rules, Documentation, Testing (context-dependent)
- **Low Priority**: Agent Interaction Unit Test (optional)

## Workflow Examples

### Example 1: Feature Implementation

```
User: "Implement user authentication with tests and documentation"

Orchestrator Workflow:
1. Plan Mode: Create technical plan (Shift + Tab)
2. Delegate to RAG Specialist: Analyze authentication patterns
3. Delegate to Memory Specialist: Retrieve previous auth work
4. Delegate to Implementation: Create auth code (parallel worktree)
5. Delegate to Testing Specialist: Create tests (parallel worktree)
6. Delegate to Critical Thinking Specialist: Validate implementation
7. Delegate to Documentation Specialist: Generate docs
8. Synthesize: Combine all results
9. Apply: Merge worktrees
```

```
4. Delegate to Testing Specialist: Verify test coverage
```


## Configuration

### Framework Settings

Configure framework in `settings/global-settings.json`:

```json
{
  "agentic_rules_framework": {
    "rule_categories": {
      "memory_rules": { "enabled": true, "priority": "medium" },
      "critical_thinking_rules": { "enabled": true, "priority": "high" },
      "rag_rules": { "enabled": true, "priority": "high" }
    }
  }
}
```

### Cursor 2.0 Adapter

Adapter configuration in `settings/cursor-2.0-multi-agent-adapter.json`:

```json
{
  "cursor_2_0": {
    "multi_agent_support": true,
    "nightly_channel_required": true,
    "composer_mode": "agent",
    "features": {
      "orchestrator": { "enabled": true },
      "sub_agents": { "enabled": true },
      "parallel_execution": { "enabled": true },
      "best_of_n_strategy": { "enabled": true }
    }
  }
}
```

## Troubleshooting

### Agents Not Available

- **Check Nightly Channel**: Ensure Cursor is on Nightly channel
- **Verify Agent Mode**: Composer must be in Agent mode
- **Generate Config Files**: Use `setup.html` to generate `.cursor/agents/*.md` files if they don't exist
- **Verify Files**: Ensure generated files are copied to `.cursor/agents/` directory in your project

### Framework Not Loading

- **Check Initialization**: Verify `.agentic_initialized` marker exists
- **Verify Bootstrap**: Check `bootstrap.json` is valid
- **Check Settings**: Verify `settings/global-settings.json` exists

### Parallel Execution Issues

- **Check Git**: Verify Git is initialized in project
- **Worktree Status**: Run `git worktree list` to check worktrees
- **Permissions**: Ensure write permissions for worktrees

### Quality Issues

- **Enable Critical Thinking**: Ensure Critical Thinking Rules enabled
- **Check Validation**: Review Critical Thinking Specialist outputs
- **Framework Logs**: Check framework logs for validation issues

## Advanced Features

### Custom Agent Creation

While the standard agents are auto-generated by `setup.html`, you can create custom agents:

1. **Generate Base Configuration**: Use `setup.html` to generate standard agents first
2. **Create Custom Agent**: Add your custom agent file to `.cursor/agents/`:

```markdown
---
name: CustomAgent
description: Your custom agent description
tools: [read_file, write, grep]
framework_integration: memory-rules
---

# Custom Agent

Your agent implementation...
```

**Note**: Custom agents should follow the same format as generated agents and integrate with framework rules as needed.

### Rule Customization

Customize framework rules in module `settings.json` files:

- `modules/memory-rules/settings.json`
- `modules/rag-rules/settings.json`
- `modules/critical-thinking-rules/settings.json`

### Integration Patterns

Extend framework integration:

1. **New Rule Modules**: Add to `modules/` directory
2. **Agent Configurations**: Generate via `setup.html` or manually add to `.cursor/agents/`
3. **Bootstrap Updates**: Update `bootstrap.json` for new rules
4. **Setup.html Updates**: Update `generate_simple_setup.py` to include new agents in generation
