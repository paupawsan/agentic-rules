# 🐛 トラブルシューティングガイド (日本語)

## 一般的な問題と解決策

### 🔄 **エディタがAGENTS.mdを自動読み込みしない**

**問題**: Cursor、VSCode、その他のエディタが`AGENTS.md`、`GEMINI.md`、または`CLAUDE.md`を自動的にエージェントのコンテキストに読み込めず、ルールが有効にならない。

**症状**:
- エージェントがルールが有効でないかのように動作
- メモリ、RAG、または批判的思考機能が動作しない
- セッション間でプロジェクトコンテキストが失われる

**解決策**: ルールファイルを手動でプロンプトに読み込む。

#### クイック手動読み込みプロンプト

**オプション1: ファイルコンテンツを貼り付け**
```
これらの構造化されたルールに従ってインタラクションを行ってください。このAGENTS.mdファイルからすべてのルールを読み込んで適用してください：

[ここにAGENTS.mdファイルのコンテンツ全体を貼り付け]

これらのルールを即座に適用し、会話全体を通して維持してください。理解し、これらのガイドラインに従うことを確認してください。
```

**オプション2: ファイルパスを提供**
```
これらの構造化されたルールに従ってインタラクションを行ってください。このパスにあるファイルからすべてのルールを読み込んで適用してください：

[path/to/your/project/]AGENTS.md

これらのルールを即座に適用し、会話全体を通して維持してください。理解し、これらのガイドラインに従うことを確認してください。
```

**代替: Cursor @ リファレンス**
```
これらの構造化されたルールに従ってインタラクションを行ってください。このファイルからすべてのルールを読み込んで適用してください：

@AGENTS.md

これらのルールを即座に適用し、会話全体を通して維持してください。理解し、これらのガイドラインに従うことを確認してください。
```

**💡 ヒント:** Cursorでは`@filename`を使用 - 自動的に正しいファイルパスに解決されます！

### 🔍 **メモリルールが動作しない**

**問題**: メモリ機能が有効でないか、情報を保持していない。

**症状**:
- エージェントが以前の会話を記憶しない
- セッション間でプロジェクトコンテキストが失われる
- 学習機能が時間とともに改善しない

**トラブルシューティングステップ**:

1. **メモリルールが有効化されているか確認**:
   - `AGENTS.md`、`GEMINI.md`、または`CLAUDE.md`にメモリルールが含まれていることを確認
   - 設定でメモリカテゴリが無効化されていないことを確認

2. **メモリストレージを検証**:
   - 期待される場所にメモリファイルが作成されているか確認
   - `memory-rules/`ディレクトリとサブディレクトリを確認

3. **メモリ保持をテスト**:
   - 質問: `"以前の会話について何を覚えていますか？"`
   - メモリルールドキュメントの検証プロンプトを試す

### 📚 **RAGルールが情報を処理しない**

**問題**: 情報処理と検索機能が動作しない。

**症状**:
- エージェントが提供されたコンテキストを効果的に使用しない
- 関連性スコアリングがおかしい
- 読み取り戦略が適用されない

**解決策**:

1. **RAGルール有効化を検証**:
   - RAGルールがアクティブなルールファイルに含まれていることを確認
   - RAG設定が適切に構成されていることを確認

### 🤔 **批判的思考ルールが適用されない**

**問題**: 体系的推論とエラー防止機能が有効でない。

**症状**:
- エージェントが検証なしで仮定を行う
- エラーパターンが防止されない
- 証拠ベースの推論に従わない

**解決策**:

1. **批判的思考ルールのステータスを確認**:
   - ルールがアクティブな構成に存在することを確認
   - 批判的思考カテゴリが有効化されていることを確認

### 🔧 **セットアップと構成の問題**

#### **Setup.pyが実行できない**
**問題**: `python setup.py`がエラーに遭遇。

**一般的な問題**:
- Pythonバージョン互換性
- 依存関係の欠如
- 権限の問題

**解決策**:
```bash
# Pythonバージョンを確認
python --version

# 正しいディレクトリにいることを確認
cd /path/to/agentic-rules

# python3で実行を試す
python3 setup.py
```

#### **Setup.htmlが読み込めない**
**問題**: Webインターフェースが適切に動作しない。

**解決策**:
- ブラウザで直接`setup.html`を開く
- ファイルアクセス付き拡張モードで`python setup-launcher.py`を使用
- JavaScriptエラーの場合はブラウザコンソールを確認

### 🌐 **プラットフォーム固有の問題**

#### **Cursor IDEの問題**
- **コンテキスト読み込み**: 上記のように手動読み込みが必要になる場合あり
- **ファイル監視**: ファイル変更が検出されない場合はCursorを再起動
- **拡張競合**: 干渉する可能性のある他のAI拡張を無効化

#### **VSCodeの問題**
- **コンテキスト制限**: 非常に長いルールファイルがトークン制限を超える可能性
- **拡張競合**: 競合するAI拡張を確認
- **ファイル関連付け**: `.md`ファイルが適切に関連付けられていることを確認

### 🚨 **緊急手動オーバーライド**

自動システムがすべて失敗した場合、この包括的な手動読み込みプロンプトを使用：

```
EMERGENCY MANUAL RULES ACTIVATION

包括的なAIエージェントルールに従う必要があります。これはデフォルト動作をオーバーライドします。

CORE RULES (これらすべてを読み込み):

[ここにAGENTS.md/GEMINI.md/CLAUDE.mdのコンテンツ全体を貼り付け]

MEMORY RULES:
[ここにMEMORY-RULES.mdのコンテンツ全体を貼り付け]

RAG RULES:
[ここにRAG-RULES.mdのコンテンツ全体を貼り付け]

CRITICAL THINKING RULES:
[ここにCRITICAL-THINKING-RULES.mdのコンテンツ全体を貼り付け]

これらすべてのルールを即座に適用し、一緒に働く期間全体を通して維持してください。各ルールカテゴリを理解し、一貫して従うことを確認してください。
```

### 📞 **ヘルプを得る**

問題が続く場合：

1. **ログを確認**: エディタのコンソールまたはターミナルでエラーメッセージを確認
2. **ファイルを検証**: 生成されたルールファイルが破損していないことを確認
3. **個別にテスト**: 各ルールシステムを個別に有効化して問題を分離
4. **問題を報告**: プロジェクトリポジトリで問題を開き、以下の情報を含む：
   - エディタとバージョン
   - 具体的なエラーメッセージ
   - 問題を再現する手順
   - オペレーティングシステム
