# 🤖 Agentic Rules Framework コアルール (日本語)

## ドキュメントの目的

**🎯 主要な対象読者**: フレームワークインテグレーター、システムアーキテクト、エージェント開発者

**📋 目的**: このドキュメントでは、フレームワークの原則と高レベル概要を実装する前に理解するための包括的な概要を提供します。フレームワークの哲学、構造、実装アプローチを理解するためのリファレンスとして機能します。

---

## 概要

Agentic Rules Frameworkは、構造化されたルールを通じてAIエージェントの機能を強化する包括的で自己完結型のシステムを提供します。このフレームワークは以下のように設計されています：

- **🔌 プラグアンドプレイ**: エージェントの動作を変更せずにルールを有効/無効化
- **📝 アルゴリズム中心**: ハードリミットではなく、行動アルゴリズムとしてルールを表現
- **🌐 マルチプラットフォーム**: Cursor、VSCode、CIシステム、カスタム実装で動作
- **🛠️ ツール非依存**: エージェントは利用可能なツールを使用してルール要件を実装
- **📦 自己完結型**: 埋め込み設定付き単一HTMLファイル（オフライン動作）
- **🌍 多言語**: 18以上の言語でローカライズされたルールテンプレートに対応
- **👀 透明性**: すべてのルール適用がアクセス可能な形式でログ記録

## フレームワークアーキテクチャ

### コアコンポーネント

#### 🎯 Webインターフェース (`setup.html`)
- **主要ツール**: ダブルクリックで開く、インストール不要
- **自己完結型**: 埋め込み設定、オフライン動作
- **多言語**: 自動ローカライズ（EN/JA/ID）
- **設定生成**: 簡単導入のためのコピー＆ペーストインターフェース
- **自動再ローカライズ**: 言語変更時に結果が更新

#### ⚙️ Bootstrapシステム (`bootstrap.json`)
- **オーケストレーションツール**: マシン読み取り可能なフレームワーク設定
- **読み込み順序**: 優先順位ベースのルール有効化
- **相互接続**: ルール間のデータフロー
- **プラットフォームアダプター**: 環境固有の適応
- **検証**: フレームワーク整合性チェック

#### 📚 ルールモジュール
- **メモリールール**: 永続的なコンテキストと知識ストレージ
- **RAGルール**: 情報処理とコンテキスト最適化
- **批判的思考ルール**: 推論検証とエラー防止

### 統合モデル

#### ルールごとの分散アーキテクチャ
- **Bootstrap設定**: `bootstrap.json`がオーケストレーションを提供
- **ルールアルゴリズム**: 個別の`AGENTS.md`ファイルに具体的な実装を含む
- **生成された統合**: Webインターフェースが有効化されたルールの`AGENTS.md`ファイルを生成
- **プラットフォーム適応**: 自動環境検出と設定

## フレームワーク原則

### 1. 🔌 プラグアンドプレイアーキテクチャ
ルールはWebインターフェースを通じて有効化/無効化できます。フレームワークは中立性を維持し、既存のエージェント操作に干渉しません。

### 2. 📝 アルゴリズム中心
ルールは実行プロセスと動作アルゴリズムを記述しますが、具体的な実装詳細を指定しません。エージェントは利用可能なツールを使用してこれらのアルゴリズムを実行する自由があります。

### 3. 🛠️ ツール独立性
エージェントは独自の機能と利用可能なツールを使用してルールアルゴリズムを実装します。フレームワークは「何をするか」を記述し、「どのようにするか」を指定しません。

### 4. 👀 透明性
すべてのルール適用、決定、プロセスはアクセス可能なマークダウン形式でログ記録されます。ユーザーはルールがエージェントの動作にどのように影響したかを確認できます。

### 5. 🏗️ フレームワーク分離
フレームワークは開発ツールのみです。決してフレームワークファイルをユーザープロジェクトのコードベースに含めません。すべての統合は生成された`AGENTS.md`ファイルを通じて行われます。

## 🎯 AIルールカテゴリ

### 🧠 メモリールール (優先度: 中)
**アルゴリズムファイル**: [`memory-rules/MEMORY-RULES.md`](memory-rules/MEMORY-RULES.md)
**設定**: [`memory-rules/settings.json`](memory-rules/settings.json)
**生成された統合**: [`memory-rules/AGENTS.md`](memory-rules/AGENTS.md) (Webインターフェースで生成)

**機能**:
- マークダウンベースの構造化知識ストレージ
- コンテキスト対応のキャプチャと検索システム
- パターン認識と分類化
- 適応型保持ポリシー
- セッション間永続メモリ

**統合要件**:
- 有効なストレージパス設定が必要
- 有効時にすべてのメモリアルゴリズムを実装する必要
- RAGルールと統合して強化されたコンテキストを提供

### 🔍 RAGルール (優先度: 高)
**アルゴリズムファイル**: [`rag-rules/RAG-RULES.md`](rag-rules/RAG-RULES.md)
**設定**: [`rag-rules/settings.json`](rag-rules/settings.json)
**生成された統合**: [`rag-rules/AGENTS.md`](rag-rules/AGENTS.md) (Webインターフェースで生成)

**機能**:
- 階層的ドキュメント読み取りと分析
- 関連性ベースのコンテキスト最適化
- マルチ言語コンテンツ処理
- LLM効率のためのコンテキストウィンドウ管理
- コードベース全体のパターン認識

**統合要件**:
- 最適なコンテキスト使用に不可欠
- メモリ構築の優先順位に影響
- 関連性スコアリングアルゴリズムを実装する必要

### 🤔 批判的思考ルール (優先度: 高)
**アルゴリズムファイル**: [`critical-thinking-rules/CRITICAL-THINKING-RULES.md`](critical-thinking-rules/CRITICAL-THINKING-RULES.md)
**設定**: [`critical-thinking-rules/settings.json`](critical-thinking-rules/settings.json)
**生成された統合**: [`critical-thinking-rules/AGENTS.md`](critical-thinking-rules/AGENTS.md) (Webインターフェースで生成)

**機能**:
- 体系的な仮定挑戦
- 多ソース検証プロトコル
- エラー受容と修正メカニズム
- 論理的一貫性検証
- 根拠に基づく推論セーフガード

**統合要件**:
- **必須** 責任あるAI動作のため
- エラー受容アルゴリズムを実装する必要
- 幻覚防止に不可欠

## ⚙️ 設定システム

### グローバル設定
**ファイル**: [`settings/global-settings.json`](settings/global-settings.json)
**目的**: フレームワークコンポーネントとプラットフォーム適応のマスターコントロール
**設定方法**: Webインターフェースまたは手動編集

### ルール固有設定
**場所**: `{rule-name}-rules/settings.json`
**目的**: 個別ルールの動作詳細設定
**設定方法**: Webインターフェースで有効化時に変更

### Bootstrap設定
**ファイル**: [`bootstrap.json`](bootstrap.json)
**目的**: マシン読み取り可能なオーケストレーションとルール相互接続
**対象**: フレームワークを実装するエージェントシステム
**変更**: 高度なユーザーのみ - 読み込み順序とプラットフォームアダプターを定義

### Webインターフェース設定
**ファイル**: [`setup.html`](setup.html) (埋め込み)
**目的**: ユーザーフレンドリーな設定インターフェース
**機能**: 言語選択、ルール有効化、設定生成
**出力**: 有効化されたルールの`AGENTS.md`ファイルを生成

## 🚀 実装ワークフロー

### 非技術ユーザー向け（クイックセットアップ）
1. **ダウンロード** コンピュータにフレームワークファイルをダウンロード
2. **ダブルクリック** `setup.html`でWebインターフェースを開く
3. **言語を選択** 🇺🇸 English / 🇯🇵 日本語 / 🇮🇩 Bahasa Indonesia
4. **有効化するルールを選択**
5. **ルール固有の設定を構成**
6. **組み込みインターフェースを使用して設定ファイルを生成**
7. **生成されたファイルをAIプロジェクトのルートにコピー＆ペースト**
8. **完了！** AIエージェントが強化された動作を使用できるようになりました

### フレームワークインテグレーター向け
1. **Webインターフェース**: `setup.html`でガイド付き設定
2. **原則のレビュー**: このドキュメントでフレームワーク哲学を読む
3. **プラットフォーム設定**: [`settings/global-settings.json`](settings/global-settings.json)で環境を設定
4. **ルール統合**: 有効化された各ルールの生成された`AGENTS.md`ファイルを読み込む
5. **Bootstrap読み込み**: [`bootstrap.json`](bootstrap.json)でオーケストレージンロジックを読む
6. **操作の監視**: 生成されたログでルール適用追跡を確認

### エージェント開発者向け
1. **アルゴリズム学習**: MEMORY-RULES.mdなどの特定のルール文書で実装詳細を読む
2. **統合ファイル**: 有効化されたルールディレクトリから`AGENTS.md`を読み込む
3. **アルゴリズム実装**: 利用可能なツールを使用してルールアルゴリズムを実行
4. **透明性ログ**: すべてのルール適用と決定プロセスをログ
5. **Bootstrap準拠**: [`bootstrap.json`](bootstrap.json)のオーケストレーション要件を尊重
6. **プラットフォーム適応**: 環境固有動作のためのプラットフォームアダプターを使用

### システムアーキテクト向け
1. **フレームワーク設計**: 分散型ルールごとの統合モデルを理解
2. **プラットフォーム計画**: `bootstrap.json`でプラットフォームアダプターを設定
3. **ルールオーケストレーション**: 最適なルール相互作用パターンを設計
4. **パフォーマンスチューニング**: 特定のユースケースと規模に合わせて設定を調整
5. **監視セットアップ**: ログとコンプライアンス監視手順を確立

## ⚠️ 重要な統合注意事項

### フレームワーク分離
**必須**: Agentic-rulesフレームワークは開発ツールのみです。決してユーザープロジェクトのコードベースにフレームワークファイルを含めません。

### 分散統合
**必須**: エージェントは個別のルールディレクトリからの`AGENTS.md`ファイル（Webインターフェースで生成）を読み込む必要があります。フレームワークファイルを直接読み込まない。

### Bootstrap準拠
**重要**: エージェントは[`bootstrap.json`](bootstrap.json)設定を尊重し、有効化されたルールのみを実行する必要があります。

### アクティブ実装
**必須**: 設定だけでは不十分です。エージェントはランタイム環境で実際にすべてのルールアルゴリズムを実装して実行する必要があります。
