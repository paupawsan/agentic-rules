# 👥 ユーザーガイド - Agentic Rules Framework (日本語)

## 誰でも利用可能（技術知識不要！）

**技術知識は一切不要！** このガイドでは、シンプルなダブルクリックでAIエージェントルールを設定する方法を説明します。

## 🚀 初回セットアップ - 完全なプロセス

### ⚠️ **重要: この順序に従ってください！**

**setup.htmlを最初に完了してから、ブートストラップ初期化を実行してください。ウェブインターフェースはブートストラップが必要とするファイルを生成します。**

---

### **フェーズ1: ウェブ設定（最初に必須）**

#### **ステップ1: セットアップのダウンロードと起動**
1. **GitHubからダウンロード** `agentic-rules`フレームワークファイルをダウンロード
2. **解凍** ファイルをコンピュータの任意のフォルダに解凍
3. **ダブルクリック** `setup.html`でウェブインターフェースを開く

#### **ステップ2: ルールの設定**
1. **言語を選択** サポートされている言語から選択
2. **ルールを選択** 有効にしたいルールを選択：
   - 🧠 **メモリルール**: 会話間でコンテキストを記憶
   - 📚 **RAGルール**: 関連情報をより良く検索・使用
   - 🤔 **批判的思考ルール**: より慎重で正確になる
3. **「設定ファイルを生成」をクリック**

> 💡 **これによりブートストラップが必要とするルールファイルと構成が作成されます！**

---

### **フェーズ2: ブートストラップ初期化（setup.htmlの後）**

#### **ステップ3: フレームワークの初期化**
AIエージェントに伝える：**`Agentic rules systemを初期化`** または **`Setup agentic rules bootstrap`**

#### **ステップ4: ブートストラップ権限の付与**
「*Bootstrap configuration not found. Initialize Agentic Rules Framework? (y/n)*」というプロンプトが表示されたら：
- 入力: `y` とEnterキーを押す

#### **ステップ5: ユーザー同意の確認**
「*The Agentic Rules Framework enhances AI behavior... Do you want to enable this framework? (y/n)*」というプロンプトが表示されたら：
- 入力: `y` とEnterキーを押す

#### **ステップ6: フレームワーク設定の確認**
選択したルールの設定を確認。setup.htmlでの選択と一致する必要があります。

#### **ステップ7: フレームワークがアクティブ！**
✅ **セットアップ完了** - ウェブ設定とブートストラップ初期化の両方が完了
✅ **ルール設定済み** - 選択した機能がアクティブ
✅ **使用準備完了** - 拡張AI機能が自動的に利用可能

---

## 🚀 フレームワーク機能の使用

## 🚀 クイックスタート（ダブルクリック方式）

### ステップ1: フレームワークのダウンロード
1. GitHubから`agentic-rules`フレームワークファイルをダウンロード
2. ダウンロードしたファイルをコンピュータの任意のフォルダに解凍

### ステップ2: セットアップの起動
1. 解凍したフォルダ内の**`setup.html`**を**ダブルクリック**
2. デフォルトのウェブブラウザが自動的に開きます

### ステップ3: ルールの設定
1. **言語を選択**: 公式サポート言語から選択
   - 🇺🇸 **英語**（主要言語）
   - 🇯🇵 **日本語**
   - 🇮🇩 **インドネシア語**

   > 💡 **プラグイン開発者向け**: カスタムプラグインを追加言語（🇩🇪 ドイツ語、🇫🇷 フランス語、🇪🇸 スペイン語など）で作成したい場合は、[拡張マニュアル](EXTENSION-MANUAL.ja.md)で`generate_plugin_scaffold.py`の多言語テンプレートサポートを使用する方法の技術詳細を参照してください。
2. **ルールを選択**: 必要なAI動作のチェックボックスをオン：
   - 🧠 **メモリルール**: 会話間でコンテキストを記憶するのに役立つ
   - 📚 **RAGルール**: AIが関連情報を検索・使用する能力を向上
   - 🤔 **批判的思考ルール**: AIをより慎重で正確にする

### ステップ4: 設定ファイルの生成
1. **「設定ファイルを生成」**をクリック
2. インターフェースがあなた専用のAIルールを作成します

### ステップ5: ファイルの保存
ファイルを保存する方法を選択：

- **💾 保存**: 保存場所を正確に選択できるファイルブラウザを開く
- **📥 ダウンロード**: ファイルを直接ダウンロードフォルダにダウンロード
- **📋 コピー**: 手動保存のためにクリップボードにコンテンツをコピー

### ステップ6: AIエージェントで使用
1. 生成されたファイルをAIプロジェクトのルートディレクトリにコピー
2. AIエージェントがルールファイルを読み込むように設定
3. これでAIエージェントが強化された機能を備えました！

```
│   ├── rag-rules/        # RAGシステムルール
│   │   ├── AGENTS.md     # RAG固有ルール
│   │   └── settings.json # RAG設定
│   └── critical-thinking-rules/  # 批判的思考ルール
│       ├── AGENTS.md     # 思考固有ルール
│       └── settings.json # 思考設定
```

## 🎯 各ルールの機能

### 🧠 メモリルール
- 会話間で重要な情報を記憶
- セッション間でコンテキストを維持
- 過去のインタラクションから学習

### 📚 RAGルール
- ファイルから関連情報を検索
- AIが情報を読んで処理する方法を最適化
- より良いコンテキストで応答精度を向上

### 🤔 批判的思考ルール
- AIが回答を二重チェック
- 間違いと「幻覚」を減少
- より慎重で正確な応答を促進

### 🧠 Knowledge Graph (KG) インテリジェンス
- **自動学習**: AIがプロジェクトの知識マップを構築
- **スマート接続**: コード、ファイル、概念間の関係を発見
- **強化された理解**: コードベースに関する深い洞察を提供
- **文脈的メモリ**: プロジェクトのさまざまな部分がどのように関連しているかを記憶

## 🧠 Knowledge Graphs が AI エクスペリエンスをどのように向上させるか

### Knowledge Graphs とは何ですか？

Knowledge Graphs (KG) は、AI が自動的に構築するインテリジェントなマップです。KG はプロジェクトのさまざまな部分間に視覚的な接続を作成し、人間の専門家のように AI がコードベースを「理解」するようにします。

```
🎯 例: AI がこのコードベースを分析し、接続を作成:

コードファイル ←→ 関数 ←→ 依存関係 ←→ 機能
    ↓         ↓         ↓         ↓
"main.py" ←→ "process_data()" ←→ "pandas" ←→ "データ処理"
"utils.py" ←→ "validate_input()" ←→ "None" ←→ "入力検証"
"config.py" ←→ "load_settings()" ←→ "json" ←→ "設定"
```

### KG 構築が自動的にどのように動作するか

**🤖 単一エージェントアーキテクチャ**: AI エージェントが自身のインテリジェンス + フレームワークアルゴリズムを使用して KG を構築

```
ユーザーがプロジェクトで作業 → AI がコードを分析 → KG 構築トリガー → 知識マップ作成
        ↓                              ↓                              ↓                    ↓
   "このファイルを分析"           "エンティティを抽出"              "関係を発見"    "接続を保存"
```

**⚡ バックグラウンド処理**: 作業中に KG 構築が隠れて発生:

```
メインバーサーション: "このプロジェクトを理解するのを手伝って"
    ↓
AI がすぐに応答 + バックグラウンドで KG 構築を開始:
├── 🔍 エンティティ発見 (関数、クラス、ファイル)
├── 🔗 関係マッピング (依存関係、呼び出し、インポート)
├── 🏗️ グラフ構築 (関連コンポーネントを接続)
├── 💾 メモリ保存 (将来の使用のために保存)
└── ⚡ クエリ準備 (複雑な質問の準備完了)
```

### 得られる実用的利点

#### **1. よりスマートなコード説明**
```
❌ KG なし: "この関数はユーザー入力を処理します"
✅ KG あり: "validate_input() はユーザー入力を処理し、検証のために error_handler() と接続され、
              main.py の process_user_request() で使用され、データサニタイズを処理"
```

#### **2. インテリジェントなプロジェクトナビゲーション**
```
ユーザー: "認証システムはどう動作しますか？"
KG 強化 AI: 接続されたすべてのコンポーネントを含む完全な認証フローを表示
```

#### **3. 依存関係の理解**
```
ユーザー: "このデータベース設定を変更するとどうなりますか？"
KG 強化 AI: "影響: user_auth() → db_connection() → data_validator() → api_response()"
```

#### **4. パターン認識**
```
ユーザー: "私のユーザーログインシステムと似ていますか？"
KG 強化 AI: "あなたのパターン: login_flow() → validate_creds() → create_session() に一致"
```

### KG が最も役立つ場合

#### **🚀 最適な用途:**
- **大規模コードベース**: 複雑なプロジェクト構造の理解
- **チームコラボレーション**: 新しい開発者のオンボーディング
- **アーキテクチャ決定**: 機能追加の計画
- **デバッグ**: 接続されたコンポーネントを介した問題追跡
- **リファクタリング**: コード変更の影響理解

#### **📈 実行可能な高度なクエリ:**
```
"ユーザーデータを処理するすべての関数を表示"
"支払いシステムに依存するコンポーネントは？"
"アプリケーションを介したエラー処理の流れは？"
"新しい機能に似たパターンを見つける"
"このデータベーススキーマを変更すると何が壊れますか？"
```

### KG 構築設定

**上級ユーザー向け**: `modules/rag-rules/settings.json` で KG 動作を制御可能:

```json
{
  "kg_background_processing": {
    "enabled": true,              // 自動 KG 構築を有効化
    "separate_conversation": true, // 作業を中断しない
    "minimal_construction": true,  // 必要なものだけ構築
    "project_analysis_only": true, // 現在のプロジェクトに範囲限定
    "batch_processing": true,      // 効率的に処理
    "no_user_waiting": true        // 常に即時応答
  }
}
```

### プライバシー & パフォーマンス

**🛡️ データはプライベートに保たれます:**
- KG はローカルファイルからのみ構築
- 外部サーバーにデータ送信なし
- 知識はプロジェクト環境内に留まる

**⚡ パフォーマンス最適化:**
- バックグラウンド処理で速度低下防止
- 増分更新 (変更されたもののみ処理)
- 頻繁にアクセスされるコンポーネントのスマートキャッシュ
- 最小限のメモリフットプリント

### KG インテリジェンスを使い始める

**🎯 シンプルに開始:**
1. **RAG Rules を有効化** setup.html で (KG 機能を含む)
2. **通常通り作業** - KG がバックグラウンドで自動構築
3. **より深い質問をする** - AI がプロジェクト関係を理解
4. **強化された応答を体験** - より正確で文脈的な回答

**それだけです！** 自動 KG 構築により AI がコードベースについて大幅にスマートになります。🧠✨

## 🔧 トラブルシューティング

### セットアップが開かない場合
- `setup.html`を右クリックして「プログラムから開く」でウェブブラウザを選択
- ZIPアーカイブ内ではなく、解凍したフォルダからファイルを開いていることを確認

### ファイルが保存できない場合
- ブラウザにファイル保存の権限があることを確認
- まず**📥 ダウンロード**オプションを試して、手動でファイルを移動

### 言語が変更されない場合
- 言語変更後にページを更新
- 翻訳が表示されない場合はブラウザのキャッシュをクリア

## 💡 ヒント

- **シンプルに始める**: まず1-2個のルールで動作を確認
- **段階的にテスト**: まず簡単なタスクで新しいルール付きのAIエージェントを試す
- **まずバックアップ**: 新しいルールを追加する前に既存のAI設定を保存
- **実験**: 異なるルールの組み合わせが異なるタイプのタスクで効果的
