## Documentation Philosophy

- **Framework docs** (README, CORE-RULES) are for humans understanding the system
- **Rule docs** (RULES.md files) are algorithm specifications for agents
- **Configuration** (JSON files) is machine-readable setup
- **Extension manual** provides step-by-step human guidance for development

## Contributing to Documentation

When adding new rules or features:

1. Update relevant documentation files
2. Add entries to this index
3. Follow the established format and structure
4. Include examples and troubleshooting
